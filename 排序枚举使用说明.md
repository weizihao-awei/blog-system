# 文章排序枚举使用说明

## 概述

本次修改实现了基于创建时间和更新时间的排序功能，通过枚举类型管理，提供更安全和规范的排序方式。

## 修改内容

### 1. 新增枚举类 `ArticleOrderEnum`

**文件路径**: `src/main/java/com/ykw/blog_system/enums/ArticleOrderEnum.java`

**支持的排序方式**:

| 枚举值 | 接口传参值 | 字段名 | 排序方向 | 说明 |
|--------|-----------|--------|---------|------|
| `CREATE_TIME_ASC` | `create_time_asc` | createTime | 升序 | 按创建时间升序（最早发布） |
| `CREATE_TIME_DESC` | `create_time_desc` | createTime | 降序 | 按创建时间降序（最新发布，默认） |
| `UPDATE_TIME_ASC` | `update_time_asc` | updateTime | 升序 | 按更新时间升序（最早编辑） |
| `UPDATE_TIME_DESC` | `update_time_desc` | updateTime | 降序 | 按更新时间降序（最新编辑） |

**特性**:
- 支持大小写不敏感的参数解析
- 提供 `fromValue()` 方法从字符串转换为枚举
- 提供 `isSupported()` 方法检查是否支持某排序值
- 包含字段名和排序方向信息

### 2. 修改 `ArticleQueryDTO`

**文件路径**: `src/main/java/com/ykw/blog_system/dto/ArticleQueryDTO.java`

**修改前**:
```java
private String orderBy = "latest";
```

**修改后**:
```java
private ArticleOrderEnum orderBy = ArticleOrderEnum.CREATE_TIME_DESC;
```

**说明**:
- 将 `orderBy` 字段从 `String` 类型改为 `ArticleOrderEnum` 枚举类型
- 默认值从 `"latest"` 改为 `ArticleOrderEnum.CREATE_TIME_DESC`（创建时间降序）

### 3. 修改 `ArticleServiceImpl.queryArticles()`

**文件路径**: `src/main/java/com/ykw/blog_system/service/impl/ArticleServiceImpl.java`

**主要改动**:

1. **导入枚举类**:
```java
import com.ykw.blog_system.enums.ArticleOrderEnum;
```

2. **更新排序逻辑**:
```java
// 根据枚举排序
ArticleOrderEnum orderEnum = queryDTO.getOrderBy();
if (orderEnum != null) {
    if (orderEnum.isAscending()) {
        wrapper.orderByAsc(getFieldMapper(orderEnum.getFieldName()));
    } else {
        wrapper.orderByDesc(getFieldMapper(orderEnum.getFieldName()));
    }
} else {
    // 默认排序：创建时间降序
    wrapper.orderByDesc(Article::getCreateTime);
}
```

3. **添加辅助方法 `getFieldMapper()`**:
```java
@SuppressWarnings("unchecked")
private <R extends Comparable<? super R>> SFunction<Article, R> getFieldMapper(String fieldName) {
    switch (fieldName) {
        case "createTime":
            return (SFunction<Article, R>) Article::getCreateTime;
        case "updateTime":
            return (SFunction<Article, R>) Article::getUpdateTime;
        default:
            return (SFunction<Article, R>) Article::getCreateTime;
    }
}
```

### 4. 更新 API 文档

**文件路径**: `API接口文档.md`

**更新的排序参数说明**:
```typescript
interface ArticleQueryDTO {
  pageNum?: number;      // 页码，默认 1
  pageSize?: number;    // 每页数量，默认 10
  categoryId?: number;   // 分类 ID
  tagId?: number;       // 标签 ID
  keyword?: string;      // 搜索关键词
  orderBy?: string;     // 排序方式，默认"create_time_desc"
                        // "create_time_asc" - 按创建时间升序（最早发布）
                        // "create_time_desc" - 按创建时间降序（最新发布）
                        // "update_time_asc" - 按更新时间升序（最早编辑）
                        // "update_time_desc" - 按更新时间降序（最新编辑）
}
```

## 使用示例

### 前端请求示例

#### 1. 获取最新文章（默认，按创建时间降序）
```json
POST /api/article/query
{
  "pageNum": 1,
  "pageSize": 10
}
```

#### 2. 按创建时间升序（最早发布的文章）
```json
POST /api/article/query
{
  "pageNum": 1,
  "pageSize": 10,
  "orderBy": "create_time_asc"
}
```

#### 3. 按创建时间降序（最新发布的文章）
```json
POST /api/article/query
{
  "pageNum": 1,
  "pageSize": 10,
  "orderBy": "create_time_desc"
}
```

#### 4. 按更新时间升序（最早编辑的文章）
```json
POST /api/article/query
{
  "pageNum": 1,
  "pageSize": 10,
  "orderBy": "update_time_asc"
}
```

#### 5. 按更新时间降序（最新编辑的文章）
```json
POST /api/article/query
{
  "pageNum": 1,
  "pageSize": 10,
  "orderBy": "update_time_desc"
}
```

#### 6. 组合查询（分类 + 关键字 + 排序）
```json
POST /api/article/query
{
  "pageNum": 1,
  "pageSize": 10,
  "categoryId": 1,
  "keyword": "Spring Boot",
  "orderBy": "create_time_asc"
}
```

## 技术要点

### 1. 类型安全
- 使用枚举替代字符串，提供编译期类型检查
- 避免拼写错误导致的运行时错误

### 2. 向后兼容
- 枚举的 `fromValue()` 方法支持大小写不敏感
- 对于不支持的值会返回默认值而非报错

### 3. 代码可维护性
- 排序逻辑集中在枚举类中管理
- 添加新排序方式只需修改枚举类

### 4. MyBatis-Plus 集成
- 使用 `SFunction` 实现类型安全的字段引用
- 支持动态升降序排列

## 注意事项

1. **默认行为变更**: 
   - 原来的默认排序是 `"latest"`（实际是按置顶和发布时间）
   - 现在默认排序是 `CREATE_TIME_DESC`（按创建时间降序）

2. **移除的排序方式**:
   - 原有的 `"hot"`（热门）排序已移除
   - 原有的 `"recommend"`（推荐）排序已移除
   - 如需这些功能，应使用专门的 `/api/article/hot` 和 `/api/article/recommend` 接口

3. **数据库字段**:
   - 确保 `article` 表有 `create_time` 和 `update_time` 字段
   - 这两个字段应该是 `datetime` 类型

## 后续优化建议

1. **扩展排序字段**: 如需支持其他字段排序（如浏览量、点赞数），可在枚举中添加：
```java
VIEW_COUNT_DESC("view_count_desc", "viewCount", false),
LIKE_COUNT_DESC("like_count_desc", "likeCount", false),
```

2. **多字段排序**: 当前只支持单字段排序，如需支持多字段排序，可修改为：
```java
private List<ArticleOrderEnum> orderByList;
```

3. **缓存优化**: 对于常用排序结果可以考虑缓存

## 总结

本次修改提供了以下四个排序选项：
- ✅ 按创建时间升序（最早发布）
- ✅ 按创建时间降序（最新发布，默认）
- ✅ 按更新时间升序（最早编辑）
- ✅ 按更新时间降序（最新编辑）

使用枚举类型提高了代码的类型安全性和可维护性，同时更新了 API 文档以便前后端协作。
