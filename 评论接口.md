# 评论模块 API 接口文档

**基础路径**: `/api/comment`

---

## 1️⃣ 获取文章评论列表

**接口**: `GET /api/comment/list/{articleId}`

**描述**: 获取指定文章的所有评论（树形结构）

**权限**: 无需认证

**路径参数**:
- `articleId` (Long): 文章 ID

**请求示例**:
```http
GET /api/comment/list/100
Authorization: Bearer <token>
```

**响应示例**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": [
    {
      "id": 1,
      "articleId": 100,
      "parentId": null,
      "userId": 10,
      "username": "张三",
      "userAvatar": "http://example.com/avatar1.jpg",
      "content": "这是第一条评论内容",
      "replyToUsername": null,
      "createTime": "2026-03-13T10:00:00",
      "children": [
        {
          "id": 2,
          "articleId": 100,
          "parentId": 1,
          "userId": 11,
          "username": "李四",
          "userAvatar": "http://example.com/avatar2.jpg",
          "content": "回复张三的评论",
          "replyToUsername": "张三",
          "createTime": "2026-03-13T10:30:00",
          "children": null
        }
      ]
    }
  ],
  "timestamp": 1710316800000
}
```

**说明**:
- 返回的数据是树形结构，通过 `parentId` 和 `children` 字段实现层级关系
- 一级评论的 `parentId` 为 `null`
- 子评论会自动嵌套在父评论的 `children` 数组中
- `replyToUsername` 表示该评论是回复哪位用户的

## 2️⃣ 创建评论

**接口**: `POST /api/comment/create`

**描述**: 发布新评论或回复评论

**权限**: 需要登录

**请求体**:
```json
{
  "articleId": 100,        // 必填，文章 ID
  "parentId": null,        // 可选，父评论 ID（回复评论时填写）
  "content": "这是评论内容"  // 必填，评论内容
}
```

**请求示例**:
```http
POST /api/comment/create
Authorization: Bearer <token>
Content-Type: application/json

{
  "articleId": 100,
  "parentId": null,
  "content": "这篇文章写得真好！"
}
```

**回复评论示例**:
```json
{
  "articleId": 100,
  "parentId": 1,
  "content": "我也这么认为！"
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "评论成功",
  "data": 15,  // 新创建的评论 ID
  "timestamp": 1710316800000
}
```

**错误响应**:
```json
{
  "code": 500,
  "message": "文章 ID 不能为空",
  "data": null,
  "timestamp": 1710316800000
}
```

## 3️⃣ 删除评论

**接口**: `DELETE /api/comment/delete/{commentId}`

**描述**: 删除指定评论及其所有子评论

**权限**: 需要登录（仅能删除自己的评论，管理员除外）

**路径参数**:
- `commentId` (Long): 评论 ID

**请求示例**:
```http
DELETE /api/comment/delete/15
Authorization: Bearer <token>
```

**响应示例**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": null,
  "timestamp": 1710316800000
}
```

**错误响应**:
```json
{
  "code": 500,
  "message": "评论不存在",
  "data": null,
  "timestamp": 1710316800000
}
```

```json
{
  "code": 500,
  "message": "无权删除此评论",
  "data": null,
  "timestamp": 1710316800000
}
```

**说明**:
- 删除评论会级联删除所有子评论
- 普通用户只能删除自己的评论
- 管理员可以删除任何评论

## 4️⃣ 更新评论状态（管理员专用）

**接口**: `PUT /api/comment/{commentId}/status/{status}`

**描述**: 审核评论（通过/不通过）

**权限**: 仅限管理员

**路径参数**:
- `commentId` (Long): 评论 ID
- `status` (Integer): 状态值
  - `1`: 通过
  - `0`: 待审核
  - `-1`: 不通过

**请求示例**:
```http
PUT /api/comment/15/status/1
Authorization: Bearer <admin_token>
```

**响应示例**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": null,
  "timestamp": 1710316800000
}
```

**错误响应**:
```json
{
  "code": 403,
  "message": "Access Denied",
  "data": null,
  "timestamp": 1710316800000
}
```

## 5️⃣ 获取评论列表（管理员分页）

**接口**: `GET /api/comment/admin-list`

**描述**: 管理员分页查看所有评论

**权限**: 仅限管理员

**查询参数**:
- `pageNum` (Integer): 页码，默认 1
- `pageSize` (Integer): 每页数量，默认 10
- `status` (Integer, 可选): 评论状态筛选

**请求示例**:
```http
GET /api/comment/admin-list?pageNum=1&pageSize=10&status=1
Authorization: Bearer <admin_token>
```

**响应示例**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "records": [
      {
        "id": 1,
        "articleId": 100,
        "parentId": null,
        "userId": 10,
        "username": "张三",
        "userAvatar": "http://example.com/avatar1.jpg",
        "content": "这是第一条评论",
        "replyToUsername": null,
        "createTime": "2026-03-13T10:00:00",
        "children": null
      },
      {
        "id": 2,
        "articleId": 100,
        "parentId": 1,
        "userId": 11,
        "username": "李四",
        "userAvatar": "http://example.com/avatar2.jpg",
        "content": "回复第一条评论",
        "replyToUsername": "张三",
        "createTime": "2026-03-13T10:30:00",
        "children": null
      }
    ],
    "total": 50,
    "pageNum": 1,
    "pageSize": 10
  },
  "timestamp": 1710316800000
}
```

**说明**:
- 此接口返回的是平铺的评论列表，不是树形结构
- 支持按状态筛选：`status=1`（通过）、`status=0`（待审核）、`status=-1`（不通过）
- 返回数据包含分页信息：`total`（总数）、`pageNum`（当前页）、`pageSize`（每页数量）

## 🔑 前端使用提示

### 1. 认证方式

除获取评论列表外，其他接口都需要在 Header 中携带 JWT Token：

```javascript
// Axios 示例
const api = axios.create({
  baseURL: '/api',
  headers: {
    'Content-Type': 'application/json'
  }
});

// 添加请求拦截器，自动注入 token
api.interceptors.request.use(config => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});
```

### 4. 错误处理

统一处理 `code !== 200` 的情况：

```javascript
// 响应拦截器
api.interceptors.response.use(response => {
  const { code, message } = response.data;
  
  if (code === 200) {
    return response.data;
  } else {
    // 显示错误提示
    ElMessage.error(message);
    return Promise.reject(new Error(message));
  }
}, error => {
  // 处理 HTTP 错误
  ElMessage.error('网络请求失败');
  return Promise.reject(error);
});
```

### 5. 数据结构说明

**CommentVO 字段说明**:
- `id`: 评论 ID
- `articleId`: 文章 ID
- `parentId`: 父评论 ID（一级评论为 null）
- `userId`: 用户 ID
- `username`: 用户名
- `userAvatar`: 用户头像 URL
- `content`: 评论内容
- `replyToUsername`: 回复的用户名（用于显示"回复 @xxx"）
- `createTime`: 创建时间
- `children`: 子评论列表（树形结构时使用）

## 📌 常见问题

### Q1: 如何实现楼中楼评论？
A: 通过设置 `parentId` 为对应评论的 ID，系统会自动构建层级关系。

### Q2: 评论有层级数限制吗？
A: 理论上没有层级限制，但建议前端展示时限制最大层级（如 5 层）以保证用户体验。

### Q3: 如何删除带子评论的评论？
A: 直接调用删除接口即可，系统会级联删除所有子评论。

### Q4: 评论需要审核吗？
A: 目前创建评论默认状态为 1（通过），如需审核可修改为 0（待审核），由管理员审核后改为 1。
