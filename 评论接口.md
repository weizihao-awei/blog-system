# 评论模块 API 接口文档

**基础路径**: `/api/comment`

---

## 1️⃣ 获取文章评论列表

**接口**: `GET /api/comment/list/{articleId}`

**描述**: 获取指定文章的评论列表（树形结构）

**权限**: 无需认证

**路径参数**:
- `articleId` (Long): 文章 ID

**请求示例**:
```http
GET /api/comment/list/100
Authorization: Bearer <token>
```

**响应示例**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": [
    {
      "id": 1,
      "articleId": 100,
      "userId": 10,
      "username": "张三",
      "userAvatar": "http://example.com/avatar1.jpg",
      "content": "这是第一条评论内容",
      "createTime": "2026-03-15T10:00:00",
      "children": [
        {
          "id": 2,
          "articleId": 100,
          "userId": 11,
          "username": "李四",
          "userAvatar": "http://example.com/avatar2.jpg",
          "content": "回复张三的评论",
          "createTime": "2026-03-15T10:30:00",
          "parentId": 1,
          "replyToUserId": 10,
          "replyToUsername": "张三",
          "replyToUserAvatar": "http://example.com/avatar1.jpg"
        }
      ]
    }
  ],
  "timestamp": 1710489600000
}
```

**说明**:
- 返回的数据是树形结构，一级评论包含 `children` 数组
- 子评论（SubCommentVO）比一级评论多了以下字段：
  - `parentId`: 父评论 ID
  - `replyToUserId`: 回复的用户 ID
  - `replyToUsername`: 回复的用户名
  - `replyToUserAvatar`: 回复用户的头像
- `createTime`: 评论创建时间，格式为 `yyyy-MM-dd HH:mm:ss`

## 2️⃣ 创建评论

**接口**: `POST /api/comment/create`

**描述**: 发布新评论或回复评论

**权限**: 需要登录

**请求体**:
```json
{
  "articleId": 100,        // 必填，文章 ID
  "parentId": null,        // 可选，父评论 ID（回复评论时填写）
  "content": "这是评论内容"  // 必填，评论内容
}
```

**请求示例**:
```http
POST /api/comment/create
Authorization: Bearer <token>
Content-Type: application/json

{
  "articleId": 100,
  "parentId": null,
  "content": "这篇文章写得真好！"
}
```

**回复评论示例**:
```json
{
  "articleId": 100,
  "parentId": 1,
  "content": "我也这么认为！"
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "评论成功",
  "data": 15,  // 新创建的评论 ID
  "timestamp": 1710489600000
}
```

**错误响应**:
```json
{
  "code": 500,
  "message": "文章 ID 不能为空",
  "data": null,
  "timestamp": 1710489600000
}
```

## 3️⃣ 删除评论

**接口**: `DELETE /api/comment/delete/{commentId}`

**描述**: 删除指定评论及其所有子评论

**权限**: 需要登录（仅能删除自己的评论）

**路径参数**:
- `commentId` (Long): 评论 ID

**请求示例**:
```http
DELETE /api/comment/delete/15
Authorization: Bearer <token>
```

**响应示例**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": null,
  "timestamp": 1710489600000
}
```

**错误响应**:
```json
{
  "code": 500,
  "message": "评论不存在",
  "data": null,
  "timestamp": 1710489600000
}
```

```json
{
  "code": 500,
  "message": "无权删除此评论",
  "data": null,
  "timestamp": 1710489600000
}
```

**说明**:
- 删除评论会级联删除所有子评论
- 普通用户只能删除自己的评论

## 🔑 前端使用提示

### 1. 认证方式

除获取评论列表外，其他接口都需要在 Header 中携带 JWT Token：

```javascript
// Axios 示例
const api = axios.create({
  baseURL: '/api',
  headers: {
    'Content-Type': 'application/json'
  }
});

// 添加请求拦截器，自动注入 token
api.interceptors.request.use(config => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});
```



### 5. 数据结构说明

**一级评论（CommentVO）字段**:
- `id`: 评论 ID (Long)
- `articleId`: 文章 ID (Long)
- `userId`: 用户 ID (Long)
- `username`: 用户名 (String)
- `userAvatar`: 用户头像 URL (String)
- `content`: 评论内容 (String)
- `createTime`: 评论创建时间 (LocalDateTime)
- `children`: 子评论列表 (List<SubCommentVO>)，仅一级评论有此字段

**子评论（SubCommentVO）字段**:
- `id`: 评论 ID (Long)
- `articleId`: 文章 ID (Long)
- `userId`: 用户 ID (Long)
- `username`: 用户名 (String)
- `userAvatar`: 用户头像 URL (String)
- `content`: 评论内容 (String)
- `createTime`: 评论创建时间 (LocalDateTime)
- `parentId`: 父评论 ID (Long)，用于标识回复哪条评论
- `replyToUserId`: 回复的用户 ID (Long)
- `replyToUsername`: 回复的用户名 (String)，用于显示"回复 @xxx"
- `replyToUserAvatar`: 回复用户的头像 URL (String)

## 📌 常见问题

### Q1: 如何实现楼中楼评论？
A: 通过设置 `parentId` 为对应评论的 ID，系统会自动构建层级关系。

### Q2: 评论有层级数限制吗？
A: 理论上没有层级限制，但建议前端展示时限制最大层级（如 5 层）以保证用户体验。

### Q3: 如何删除带子评论的评论？
A: 直接调用删除接口即可，系统会级联删除所有子评论。

### Q5: 如何显示"回复 @xxx"？
A: 子评论中的 `replyToUsername` 字段就是用来显示回复对象的用户名，例如："回复 @张三"。
